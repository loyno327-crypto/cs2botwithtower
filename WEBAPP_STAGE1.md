# Этап 1: Web App подключён к боту — что сделано и как проверить

## Что изменилось в коде
- `webserver.py` — новый файл: веб-сервер (aiohttp), отдаёт папку `webapp/`
  и эндпоинт `/api/me`, который проверяет подпись Telegram и возвращает
  баланс/уровень/XP из твоей текущей базы `database.db`.
- `bot.py` — веб-сервер теперь запускается как фоновая задача рядом с
  polling'ом бота (`asyncio.create_task(start_webserver())`).
- `config.py` — добавлена переменная `WEBAPP_URL` (адрес твоего Web App).
- `handlers/start.py` — в главном меню появляется кнопка **🎮 Играть**
  (только если `WEBAPP_URL` задан), она открывает Web App.
- `webapp/index.html` — сама страница: шапка с живым профилем (баланс,
  уровень, XP) и нижняя навигация с 5 вкладками-заглушками (реальные
  экраны — на следующих этапах).

## Как задеплоить на bothost.ru
1. Запушь изменения в тот же GitHub-репозиторий, что уже подключён к
   bothost (`git add . && git commit -m "stage 1: webapp shell" && git push`).
2. В панели bothost проект передеплоится (или нажми Deploy вручную).
3. **Узнай публичный адрес бота** в панели bothost (обычно вида
   `https://bot-XXXX.bothost.ru`).
4. В настройках проекта на bothost добавь переменную окружения:
   ```
   WEBAPP_URL = https://bot-XXXX.bothost.ru
   ```
   (тот же адрес, что в п.3 — просто без пути, на корень).
5. Перезапусти бота, чтобы переменная подхватилась.

## ⚠️ Единственный момент, который нужно проверить руками
Веб-сервер слушает порт из переменной окружения `PORT` (это стандарт для
большинства PaaS-платформ). Я не могу на 100% подтвердить, что bothost
использует именно `PORT` — в их документации явно про это не написано.

Как проверить после деплоя:
- Открой `https://bot-XXXX.bothost.ru/api/me` в браузере.
- Если видишь `{"error":"invalid_init_data"}` — всё работает, сервер
  ответил (401 — это нормально, ты же не Telegram).
- Если видишь ошибку прокси/таймаут — зайди в поддержку/чат bothost и
  спроси, на каком порту они ждут HTTP-сервер для входящих запросов, и
  пришли мне — поправлю одну строчку в `webserver.py`.

## Регистрация Web App в @BotFather (опционально, для кнопки меню слева)
Кнопка "🎮 Играть" в клавиатуре уже работает и без этого. Но если хочешь
ещё и кнопку рядом со скрепкой (Menu Button):
1. Напиши @BotFather → `/mybots` → выбери бота → **Bot Settings** →
   **Menu Button** → **Configure Menu Button**.
2. Пришли `https://bot-XXXX.bothost.ru` и текст кнопки, например `Играть`.

## Как тестировать
1. Открой бота в Telegram, `/start`.
2. Нажми **🎮 Играть**.
3. Должна открыться тёмная страница с твоим ником, балансом, уровнем и
   XP — теми же значениями, что показывает команда "💰 Баланс" в самом
   боте (значит, общая база данных подтверждена).
4. Понажимай вкладки внизу — переключаются визуально (это ещё заглушки).

## Если что-то пошло не так
- `/api/me` не отвечает вообще → см. блок про порт выше.
- Кнопка "Играть" не появилась → не задан `WEBAPP_URL`, или бот не
  перезапущен после добавления переменной.
- Открывается белый экран → проверь консоль браузера (Web App можно
  открыть и в Telegram Desktop, там есть DevTools).

Когда протестируешь и всё ок — переходим к Этапу 2 (полноценный каркас
интерфейса: реальные разделы вместо заглушек). Если захочешь что-то
поменять в идее до этого — самое время сказать.
