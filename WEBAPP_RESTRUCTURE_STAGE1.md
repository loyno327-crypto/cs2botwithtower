# Смена концепции, Этап 1: структура Web App — что сделано и что осталось

## Что сделано

- `webapp/index.html` полностью переписан под новую структуру из 5
  вкладок в 7:
  - Убрана вкладка **Играть** и весь код режима "Оборона" (Canvas-движок
    боя, путь врагов, точки застройки, попадания/броня/каска/Kill Feed,
    экран результата боя) — это было ~500 строк JS и вся связанная
    разметка/CSS. Бэкенд-эндпоинты `/api/battle/config` и
    `/api/battle/finish` в `webserver.py` не трогал — просто больше не
    вызываются с фронтенда.
  - Добавлены новые вкладки: **Игры**, **Кейсы**, **Магазин**, **Топ**
    (были только Главная/Играть/Магазин/Инвентарь/Профиль).
  - **Игры** — список из 3 карточек (Crash / Апгрейд / Слоты), каждая
    открывает свой под-экран-заглушку с описанием того, что там будет.
  - **Кейсы** — сюда перенесён существующий рендер списка кейсов
    (раньше жил во вкладке "Магазин"), API `/api/shop/cases` тот же.
  - **Магазин** — новый раздел-заглушка под эксклюзивные предметы за
    монеты (реализация — Этап 2 по новому плану).
  - **Инвентарь** — логика та же, добавлена неактивная иконка замка на
    каждом предмете — визуальный задел под защиту предметов.
  - **Профиль** — убраны строки статистики режима "Оборона" (бои,
    точность, хедшоты, рекорд волн), так как режим снят с разработки.
  - **Топ** — переключатель "Монеты / Уровень / Дорогой дроп"
    кликабельный визуально, без данных (подключение — Этап 3).
  - Нижняя навигация переверстана на 7 кнопок в один ряд (уменьшены
    иконки/шрифт подписи, раньше было 5 кнопок).
- Файл проверен: инлайн-JS проходит `node --check`, разметка содержит
  ровно 7 `view` и 7 кнопок вкладок.
- Бэкенд (`webserver.py`, `bot.py`, `handlers/*`, `database.py`) не
  менялся — это была чисто фронтенд-переверстка структуры.

## Как задеплоить

```
git add .
git commit -m "restructure stage 1: new tabs (games/cases/shop/inventory/profile/top), remove TD mode"
git push
```

Переменные окружения не менялись, ничего дополнительно настраивать не
нужно.

## Как проверить

1. Открой бота → **🎮 Играть** → должно быть 7 вкладок внизу:
   Главная / Игры / Кейсы / Магазин / Инвентарь / Профиль / Топ.
2. **Играть** как отдельной вкладки больше нет — старый режим "Оборона"
   нигде не должен всплывать.
3. **Игры** → тапни по любой из трёх карточек → открывается заглушка с
   описанием и кнопка "‹ Все игры" возвращает к списку.
4. **Кейсы** — список кейсов должен подгружаться так же, как раньше
   подгружался в "Магазине".
5. **Магазин** и **Топ** — просто заглушки, без ошибок в консоли.
6. **Инвентарь** и **Профиль** — данные те же, что были, только в
   Профиле пропали строки про "Оборону".

## Что осталось (промт для продолжения)

Ниже — спецификация оставшейся работы в том же формате, в каком была
поставлена исходная задача. Можно отдавать по частям (по одному
разделу) или целиком.

```
Продолжить разработку Telegram Web App поверх текущей структуры
(Этап 1 уже сделан: вкладки Главная/Игры/Кейсы/Магазин/Инвентарь/
Профиль/Топ реализованы, режим "Оборона" удалён).

1. ИГРЫ — перенести реальную логику вместо заглушек:
   1.1. Crash:
        - коэффициент растёт плавно в реальном времени, каждая цифра
          табло анимируется отдельно;
        - вероятность краша и сама механика полностью совпадают с
          handlers/crash.py в боте (использовать тот же бэкенд-расчёт,
          не дублировать логику на фронтенде отдельно);
        - ставка и результат синхронизируются с балансом бота
          мгновенно.
   1.2. Апгрейд:
        - перенести handlers/upgrade.py в веб-интерфейс: выбор
          предмета из инвентаря, отображение шанса, анимация процесса,
          отдельные анимации успеха и неудачи;
        - игровая логика (шансы) не меняется.
   1.3. Слоты:
        - перенести игру слотов в Web App;
        - изменить баланс: увеличить вероятность выигрыша, уменьшить
          размер выигрыша — внести изменения и в веб-версию, и в
          саму механику бота, чтобы логика была одинаковой в обоих
          местах.
   После переноса каждой игры — протестировать её отдельно перед
   переходом к следующей.

2. КЕЙСЫ — добавить дорогие кейсы:
   - у части игроков баланс превышает 300 000 монет, нужны кейсы под
     этот уровень;
   - в новых кейсах — ножи, дорогие скины, редкие предметы, ширпотреб,
     разные уровни редкости;
   - чем дороже кейс — тем выше шанс ценного предмета;
   - добавить кейсы в static/cases.json и (если нужно открытие прямо
     из Web App, а не только в боте) реализовать эндпоинт открытия.

3. МАГАЗИН — реализовать вместо заглушки:
   - полноценный магазин за игровые монеты;
   - эксклюзивные предметы, недоступные из кейсов или с крайне низким
     шансом выпадения;
   - высокая цена, рассчитанная на игроков с большим балансом;
   - ассортимент должен легко расширяться (аналогично cases.json —
     отдельный конфиг-файл, а не хардкод в коде).

4. ИНВЕНТАРЬ — защита предметов:
   - заменить неактивную иконку замка на рабочую кнопку;
   - предмет можно пометить как "сохранённый";
   - защищённый предмет нельзя продать, нельзя использовать в
     апгрейде, он не участвует в массовых операциях;
   - защиту можно снять в любой момент;
   - нужна новая колонка в БД (например inventory.is_protected) и
     проверка в местах продажи/апгрейда/массовых операций — и в
     Web App, и в самом боте.

5. ТОП — подключить реальные данные вместо заглушки:
   - три рейтинга: по количеству монет, по уровню аккаунта, по самому
     дорогому предмету в инвентаре;
   - быстрое переключение между ними (переключатель уже есть в
     интерфейсе, нужен только бэкенд — например GET /api/top?type=coins
     |level|item);
   - рейтинги считаются по общей базе бота, без отдельного хранилища.

6. ДИЗАЙН — единый тёмный игровой стиль:
   - привести все разделы к одной цветовой палитре (сейчас используется
     тёмно-зелёная тактическая тема, оставшаяся от режима "Оборона" —
     решить, сохранять её или менять на более нейтральную "казино"-тему,
     раз тема "Обороны" больше не актуальна);
   - тёмный фон, светлый текст и цифры, контрастные кнопки, карточки со
     скруглёнными углами;
   - интерфейс должен выглядеть как отдельное игровое приложение, а не
     как стандартный Telegram Mini App.

7. ФИНАЛ:
   - доработать анимации/визуальные эффекты;
   - исправить найденные в процессе ошибки;
   - финальное тестирование всех функций и синхронизации Web App с
     ботом.
```

Рекомендуемый порядок — как в оригинальном плане: сначала весь блок 1
(игры), затем 2–4 (кейсы/магазин/инвентарь), затем 5–7 (топ/дизайн/
финал). Но можно и вразнобой — просто дай знать, с какого пункта
начинать дальше.
