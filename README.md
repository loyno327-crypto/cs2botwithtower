# CS2 Case Bot

Реализовано на данный момент:

1. Регистрация пользователя при первом запуске (`/start`) — новый игрок сразу
   получает стартовый баланс `START_BALANCE` (по умолчанию 1000 монет).
2. Баланс каждого участника (хранится в базе данных `database.db`)
3. Кнопка **"💼 Работа"** — бот задаёт вопрос с 4 вариантами ответа (сейчас
   30 вопросов про CS2 в базе). Правильный ответ даёт случайную награду
   (20–100 монет), неправильный — ничего. Есть небольшая пауза (кулдаун)
   между попытками.
4. Кнопка **"🎁 Кейсы"** — 5 кейсов разной ценовой категории (от 50 до 1200
   монет), просмотр содержимого, открытие за монеты (предмет выпадает
   случайно с учётом весов редкости) и добавление в инвентарь.
5. Кнопка **"📦 Инвентарь"** — список предметов пользователя с кнопкой продажи
   у каждого. Продажа возвращает `SELL_PERCENT` (по умолчанию 70%) от цены
   предмета — специально меньше 100%, чтобы нельзя было бесконечно фармить
   баланс схемой "открыл кейс -> сразу продал" без потерь.
6. Кнопка **"🛠 Апгрейд"** — выбираешь от 1 до `MAX_UPGRADE_ITEMS` предметов
   из инвентаря и множитель (x1.2/x2/x3/x5/x10). С шансом из
   `UPGRADE_MULTIPLIERS` суммарная цена увеличивается в это число раз и все
   предметы объединяются в один новый, иначе всё сгорает (сгоревшие монеты
   уходят в общий джекпот). Матожидание каждого множителя специально меньше
   1 — комиссия дома. Та же логика доступна и в Web App (раздел «Игры» →
   «Апгрейд»), с анимацией шкалы шанса.
7. Кнопка **"⚔️ Дуэль"** — PvP на монеты. Выбираешь ставку, бот ищет игрока
   с такой же ставкой в очереди. Как только пара находится — 50/50, победитель
   забирает банк за вычетом комиссии `DUEL_HOUSE_FEE_PERCENT`.
8. Кнопка **"🏆 Топ"** — рейтинг игроков по балансу.
9. Кнопка **"🎉 Бонус"** — ежедневная награда раз в `DAILY_BONUS_COOLDOWN_HOURS`
   часов (по умолчанию 24).
10. Кнопка **"📊 Статистика"** — баланс, размер и стоимость инвентаря, самый
    дорогой предмет, статистика по "Работе" (верно/неверно), количество
    открытых кейсов, статистика дуэлей (сыграно/побед) и апгрейдов
    (удачных/сгоревших).

## Как запустить бота (пошагово)

### 1. Установи библиотеки

Открой терминал (командную строку) в папке с проектом и выполни:

```
pip install -r requirements.txt
```

### 2. Вставь токен бота

Открой файл `config.py` и замени строку:

```python
BOT_TOKEN = "ВСТАВЬ_СЮДА_СВОЙ_ТОКЕН"
```

на свой реальный токен от @BotFather, например:

```python
BOT_TOKEN = "123456789:AAExampleTokenHere"
```

### 3. Запусти бота

В терминале, находясь в папке проекта, выполни:

```
python bot.py
```

Если всё правильно — в терминале появится надпись:
`Бот запущен. Нажми Ctrl+C, чтобы остановить.`

Теперь можно писать боту в Telegram: `/start`.

Чтобы остановить бота — нажми `Ctrl+C` в терминале. Пока терминал открыт и бот
запущен, он будет отвечать. Как только закроешь терминал/выключишь компьютер —
бот перестанет отвечать (это нормально для тестового режима на своём компьютере).

## Структура проекта

```
cs2_bot/
├── bot.py                # точка входа, запуск бота
├── config.py              # токен и все игровые настройки
├── database.py             # работа с базой данных SQLite
├── requirements.txt        # список библиотек
├── database.db              # создастся автоматически при первом запуске
├── data/
│   ├── questions.json        # вопросы для "Работы" (можно редактировать/добавлять)
│   └── cases.json             # кейсы: цена, список предметов, редкость, вес дропа
└── handlers/
    ├── __init__.py
    ├── start.py               # /start, главное меню, баланс
    ├── work.py                 # логика "Работы" (викторина)
    ├── cases.py                # список кейсов, просмотр, открытие
    ├── inventory.py             # список инвентаря и продажа предметов
    ├── upgrade.py                # апгрейд предметов (риск сгореть)
    ├── pvp.py                     # PvP-дуэли на монеты
    ├── top.py                      # топ игроков и ежедневный бонус
    └── stats.py                     # статистика игрока
```

## Как добавить свои вопросы

Открой `data/questions.json` и добавь новый блок по образцу:

```json
{
  "question": "Твой вопрос?",
  "options": ["Вариант 1", "Вариант 2", "Вариант 3", "Вариант 4"],
  "correct": 0
}
```

`"correct"` — это номер правильного варианта, считая от 0 (0 = первый, 1 = второй и т.д.)

## Настройка экономики (продажа, апгрейд, дуэли, бонус)

Все цифры экономики — в одном месте, `config.py`:

```python
SELL_PERCENT = 0.7                          # % возврата при продаже предмета
UPGRADE_MULTIPLIERS = {2: 0.47, 3: 0.30, 5: 0.17, 10: 0.08}  # множитель: шанс успеха
DUEL_BET_OPTIONS = [50, 100, 250, 500, 1000]  # доступные ставки в дуэлях
DUEL_HOUSE_FEE_PERCENT = 5                    # комиссия с банка дуэли
DAILY_BONUS_MIN = 100
DAILY_BONUS_MAX = 300
DAILY_BONUS_COOLDOWN_HOURS = 24
START_BALANCE = 1000                          # стартовый баланс нового игрока
```

**Важно:** `START_BALANCE` применяется только к новым игрокам (тем, кто
нажимает `/start` впервые). Баланс тех, кто уже зарегистрирован в базе, не
меняется — это ожидаемое поведение, иначе можно было бы обнулять/накручивать
чужой прогресс, просто поменяв настройку.

## Важный нюанс про PvP

Дуэли работают через очередь ожидания в базе данных: если соперник с такой
же ставкой не найден сразу, ставка игрока замораживается и он попадает в
список ожидания (можно отменить поиск кнопкой — деньги вернутся). Бот
отправляет результат дуэли *обоим* игрокам напрямую в их личный чат — для
этого оба должны хотя бы раз написать боту `/start`.
