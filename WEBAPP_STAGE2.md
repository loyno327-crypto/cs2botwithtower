# Этап 2: реальные разделы вместо заглушек — что сделано и как проверить

## Что изменилось в коде
- `webserver.py` — добавлены три новых эндпоинта (та же логика проверки
  подписи Telegram, что и у `/api/me` из Этапа 1):
  - `GET /api/inventory` — реальный инвентарь игрока (та же таблица
    `inventory`, что и в боте, `handlers/inventory.py`).
  - `GET /api/stats` — подробная статистика (то же, что видно в боте
    через "👤 Профиль", `handlers/stats.py`): баланс, опыт, кейсы,
    дуэли, апгрейды, джекпот, лучший предмет.
  - `GET /api/shop/cases` — витрина кейсов: читает тот же
    `static/cases.json`, которым пользуется `handlers/cases.py`, и
    отдаёт укороченный список (без служебных весов дропа).
- `webapp/index.html` — страница переписана в SPA-каркас: теперь у
  каждой из 5 вкладок свой реальный экран, а не визуальная заглушка:
  - **Главная** — живые цифры (заработано всего, кейсов открыто,
    дуэлей сыграно, апгрейдов успешно) + текстовый ориентир, что будет
    на следующем этапе.
  - **Играть** — выбор стороны 🔴 Т / 🔵 CT (описан в концепции). Сам
    бой ещё не запускается — это Этап 3, выбор стороны сейчас только
    запоминается на устройстве.
  - **Магазин** — список кейсов бота (цена, мин. уровень, топ-4
    предмета из возможного дропа). Открытие кейса пока происходит в
    самом боте — прямая покупка из Web App появится на Этапе 7.
  - **Инвентарь** — настоящий список предметов игрока с редкостью
    (цвет полоски слева совпадает с эмодзи-кодировкой редкости в боте)
    и ценой, плюс сводка (кол-во предметов / суммарная стоимость).
  - **Профиль** — полная таблица статистики + карточка лучшего
    предмета в коллекции.
  - Вкладки теперь переключают реальные `<div class="view">`, каждый
    раздел подгружает данные один раз и кэширует их на время сессии
    (повторное открытие вкладки не бьёт по серверу заново).

Бэкенд по-прежнему живёт в том же процессе бота — никакого нового
деплоя или сервиса создавать не нужно.

## Как задеплоить
1. Запушь изменения в тот же репозиторий:
   ```
   git add .
   git commit -m "stage 2: real tabs (home/play/shop/inventory/profile)"
   git push
   ```
2. Дождись передеплоя на bothost (или запусти вручную).
3. Переменные окружения (`BOT_TOKEN`, `WEBAPP_URL`, `PORT`) менять не
   нужно — они уже настроены на Этапе 1.

## Как тестировать
1. Открой бота → `/start` → **🎮 Играть**.
2. **Главная** — цифры должны совпадать с тем, что видно в самом боте
   (например, "Кейсов открыто" — с тем, что ты реально открывал через
   `handlers/cases.py`).
3. **Играть** — нажми на 🔴 или 🔵 карточку, она должна подсветиться,
   кнопка снизу — поменять текст на "Сторона: … ✓".
4. **Магазин** — должны появиться все кейсы из `static/cases.json` с
   ценами и примерами предметов.
5. **Инвентарь** — если у тебя есть предметы (открывал кейсы в боте) —
   они должны появиться здесь с теми же названиями/ценами/редкостью.
   Если инвентарь пуст — увидишь дружелюбную заглушку с подсказкой.
6. **Профиль** — сверь пару цифр (баланс, уровень) с командой
   "👤 Профиль" в самом боте — должны совпадать один в один, это и есть
   подтверждение общей базы данных.
7. Переключение вкладок туда-обратно не должно вызывать повторную
   загрузку с "мельканием" — данные кэшируются после первого захода.

## Если что-то пошло не так
- Любая вкладка показывает "не удалось загрузить…" → открой Web App
  через кнопку бота, а не напрямую в браузере (без `initData` от
  Telegram сервер отвечает 401 — это ожидаемо и правильно).
- Магазин или инвентарь пустые, хотя не должны быть → проверь, что
  `static/cases.json` реально лежит рядом с `webserver.py` в
  задеплоенном проекте (структура папок не должна меняться).
- Цифры в Web App отличаются от цифр в боте → проверь, что оба
  используют один и тот же файл базы данных (см. `database.py`,
  переменная пути к `database.db`), а не два разных инстанса.

Когда протестируешь — переходим к **Этапу 3**: первый играбельный
бой (одна карта, точка, волны врагов, победа/поражение — пока без
промахов, хедшотов и брони, это Этап 4). Если захочешь скорректировать
что-то в концепции сейчас — самое время сказать, пока Этап 3 не начат.
