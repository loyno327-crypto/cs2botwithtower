# Этап 3: первый играбельный бой — что сделано и как проверить

## Что изменилось в коде
- `database.py`
  - Три новые колонки статистики: `td_battles_played`, `td_wins`,
    `td_best_wave` (префикс `td_` — Tower Defence, чтобы не путать с
    текстовыми режимами бота).
  - Новая функция `record_td_battle_result(user_id, wave_reached, won,
    reward_coins, reward_xp)` — фиксирует итог боя: считает сыгранные
    бои/победы, обновляет рекорд волны (только если новый результат
    выше), начисляет монеты через `add_balance` (тот же total_earned,
    что и раньше) и опыт через `add_xp` (тот же механизм повышения
    уровня, что и в остальном боте).
- `webserver.py`
  - `BATTLE_CONFIG` — единственный источник правды по балансу боя
    (волны, HP точки, характеристики врагов и башни, награды). Фронтенд
    ничего не хардкодит — берёт эти цифры с сервера.
  - `GET /api/battle/config` — отдаёт этот конфиг.
  - `POST /api/battle/finish` — принимает итог боя (`side`,
    `wave_reached`, `won`), сам считает награду по `BATTLE_CONFIG` (клиент
    не может подделать сумму монет, просто прислав другое число) и
    записывает результат через `record_td_battle_result`.
  - `/api/stats` теперь отдаёт и `td_battles_played` / `td_wins` /
    `td_best_wave`.
- `webapp/index.html` — вкладка **Играть** получила полноценный игровой
  цикл на Canvas (vanilla JS, без библиотек):
  - Одна карта: враги идут по фиксированной ломаной дороге к точке
    (💣 для стороны Т — бомба уже заложена, 🎯 для CT — её ещё нужно не
    дать заложить).
  - 4 слота для построек рядом с дорогой — тап по пустому слоту
    бесплатно ставит бойца (экономика/редкость/стоимость бойцов —
    Этапы 5–7, сейчас это один базовый тип для проверки цикла).
  - Башня автоматически стреляет по ближайшему врагу в радиусе действия,
    урон фиксированный — **без промахов, без хедшотов, без брони**, это
    осознанно оставлено на Этап 4.
  - 5 волн, каждая следующая — больше врагов, больше HP, чуть быстрее.
  - HP точки, если враг дошёл до конца — точка теряет здоровье; дошло
    до нуля — поражение сразу.
  - Прошёл все 5 волн — победа. Экран результата: убийства, полученные
    монеты и опыт, обновлённый рекорд по волнам.
  - После боя кэш статистики сбрасывается — на вкладках Главная/Профиль
    сразу видно актуальные цифры.

## Как задеплоить
```

git push
```
Переменные окружения не менялись — деплой как обычно, без ручных
настроек.

## Как тестировать
1. Открой Web App → вкладка **Играть**.
2. Выбери сторону (🔴 или 🔵) — кнопка снизу должна поменять текст на
   "Начать бой: …" и стать активной (жёлтой).
3. Нажми **Начать бой** — должен открыться Canvas с дорогой, точкой и
   первым врагом, который постепенно идёт к точке.
4. Тапни по пунктирному кружку у дороги — должен появиться боец (🔫),
   через мгновение он начнёт стрелять по ближайшему врагу в радиусе
   (полупрозрачный жёлтый круг вокруг бойца — это его радиус).
5. Поставь ещё 2-3 бойца (максимум 4) в разных точках дороги — волны
   должны становиться заметно сложнее (больше врагов, толще HP-полоска).
6. Дай точке "получить урон" (не ставь бойцов вовсе) — здоровье точки
   вверху должно уменьшаться, при 0 — сразу экран поражения.
7. Пройди все 5 волн (или проиграй специально) — экран результата должен
   показать волну/убийства/монеты/опыт. Проверь, что монеты и опыт
   реально добавились — открой вкладку **Профиль**, баланс/опыт должны
   вырасти на показанные в результате числа.
8. "Рекорд волн" в результате и в **Профиле** должен обновляться только
   в большую сторону (проиграй на волне 2 после победы на волне 5 —
   рекорд должен остаться 5, а не смениться на 1).
9. Кнопка **Сдаться** во время боя должна завершать бой как поражение
   на текущей волне (без прохождения её до конца).

## Известные упрощения этого этапа (осознанно, не баг)
- Бой полностью считается в браузере на устройстве игрока, сервер лишь
  принимает финальный итог. Полноценная защита от накрутки (повторная
  отправка одного и того же боя, проверка длительности и т.п.) не
  реализована — для тестового этапа это нормально, стоит добавить перед
  тем как награды станут по-настоящему ценными.
- Боец один тип, бесплатный, без брони/промахов/хедшотов у врагов —
  всё это по плану идёт на Этапах 4–7.
- Прогресс волны не сохраняется при закрытии Web App посреди боя —
  бой либо доигрывается, либо засчитывается поражением по кнопке
  "Сдаться".

## Если что-то пошло не так
- Канвас пустой/не рисуется → открой консоль браузера (Telegram Desktop
  → Web App → правый клик → Inspect, либо см. `DEPLOY_BOTHOST.md` про
  доступ к DevTools) и пришли ошибку.
- После победы/поражения баланс не изменился → проверь вкладку
  **Профиль**, возможно кэш не обновился — попробуй закрыть и снова
  открыть Web App через кнопку бота.
- "Начать бой" не реагирует → проверь, что сторона реально выбрана
  (карточка должна подсветиться цветной рамкой).

Когда протестируешь — переходим к **Этапу 4**: полноценная боевая
механика по спецификации — промах/попадание в тело или голову, броня,
каска, Kill Feed. Если захочешь что-то поменять в самом игровом цикле
(скорость волн, карта, количество слотов) — самое время сказать, пока
Этап 4 не начат: он будет достраиваться поверх именно этого цикла.
