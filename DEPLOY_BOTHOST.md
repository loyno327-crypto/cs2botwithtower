# Деплой бота на bothost.ru (бесплатно, работает без включённого ПК)

## 1. Залей код на GitHub
1. Зарегистрируйся на github.com, если ещё нет аккаунта.
2. Создай **приватный** репозиторий (например `cs2-bot`).
3. Загрузи в него все файлы из этой папки (`bot.py`, `config.py`, `database.py`,
   `handlers/`, `data/`, `requirements.txt` и т.д.).
   Проще всего — через github.com/new, кнопка "uploading an existing file",
   или через git:
   ```bash
   cd cs2_bot
   git init
   git add .
   git commit -m "init"
   git branch -M main
   git remote add origin https://github.com/ТВОЙ_НИК/cs2-bot.git
   git push -u origin main
   ```

## 2. Получи токен бота
Если ещё нет — напиши @BotFather в Telegram, команда `/newbot`, следуй
инструкциям, скопируй выданный токен (выглядит как `123456:ABC-DEF...`).

## 3. Создай проект на bothost.ru
1. Зайди на bothost.ru, зарегистрируйся (карта не нужна).
2. Создай новый проект → выбери тип "Python" → подключи GitHub-репозиторий
   `cs2-bot`.
3. В настройках проекта:
   - **Start command / команда запуска**: `python bot.py`
   - **Requirements file**: `requirements.txt` (bothost сам поставит зависимости)
   - **Environment variables / переменные окружения**: добавь
     ```
     BOT_TOKEN = твой_токен_от_BotFather
     ```
4. Запусти проект (Deploy / Start).

## 4. Проверь
Открой своего бота в Telegram и напиши `/start`. В панели bothost должны
идти логи запуска ("Бот запущен...").

## Важно про базу данных
Бот хранит данные в файле `database.db` рядом с кодом (SQLite). Уточни в
панели bothost, что диск проекта постоянный (persistent) — тогда данные
переживут перезапуск бота. Если платформа периодически "обнуляет" диск на
бесплатном тарифе — рекомендуется не полагаться на бесплатный тариф для
долгосрочного хранения важных данных, либо периодически делать бэкап
`database.db` (скачивать через панель).

## Обновление бота в будущем
Просто запушь изменения в GitHub (`git add . && git commit -m "..." && git push`) —
если включён автодеплой, bothost сам перезапустит бота с новым кодом.
